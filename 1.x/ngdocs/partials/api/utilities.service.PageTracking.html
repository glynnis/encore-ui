PageTracking
service in module utilities

Description

This is the data service that can be used in conjunction with the pagination objects to store/control page display of data tables and other items. This is intended to be used with rxPaginate.directive:rxPaginate

Methods

  • createInstance(options)

    This is used to generate the instance of the PageTracking object. It takes an optional options object, allowing you to customize the default pager behaviour.

    Parameters
    ParamTypeDetails
    optionsObject

    Configuration options for the pager

    options.itemsPerPagenumber

    The default number of items to display per page. If you choose a value * that is not in the default set ot itemsPerPage options (50, 200, 350, 500), then that value will be inserted into that list in the appropriate place

    (default: 200)

    options.itemSizeListnumber[]

    The "items per page" options to give to the user. As these * same values are used all throughout Encore, you probably should not alter them for your table.

    (default: (50, 200, 350, 500))

    options.persistItemsPerPageboolean

    Whether or not a change to this pager's itemsPerPage should be persisted globally to all other pagers

    (default: true)

    options.pagesToShownumber

    This is the number of page numbers to show in the pagination controls

    (default: 5)

    options.showAllboolean

    This is used to determine whether or not to use the pagination. If true, then all items will be displayed, i.e. pagination will not be used

    (default: false)

    Returns
    Object

    A new pager instance to be passed to the page-tracking attribute of <rx-paginate> (see rxPaginate.directive:rxPaginate)

Example

$scope.pager = PageTracking.createInstance({showAll: true, itemsPerPage: 15});

<rx-paginate page-tracking="pager"></rx-paginate>
