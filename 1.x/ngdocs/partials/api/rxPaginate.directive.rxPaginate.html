rxPaginate
directive in module rxPaginate

Description

Directive that takes in the page tracking object and outputs a page switching controller. It can be used in conjunction with the Paginate filter for UI-based pagination, or can take an optional serverInterface object if you instead intend to use a paginated server-side API

Usage

as element:
<rx-paginate
       page-tracking="{Object}"
       number-of-pages="{Number}"
       [server-interface]="{Object}"
       [filter-text]="{Object}"
       [selections]="{Object}"
       [sort-column]="{Object}"
       [sort-direction]="{Object}"
       [error-message]="{String}">
</rx-paginate>

Parameters

ParamTypeDetails
pageTrackingObject

This is the page tracking service instance to be used for this directive. See utilities.service:PageTracking

numberOfPagesNumber

This is the maximum number of pages that the page object will display at a time.

[serverInterface]Object

An object with a getItems() method. The requirements of this method are described in the rxPaginate module documentation

[filterText]Object

The model for the table filter input, if any. This directive will watch that model for changes, and request new results from the paginated API, on change

[selections]Object

The selected property of a SelectFilter instance, if one is being used. This directive will watch the filter's selections, and request new results from the paginated API, on change

[sortColumn]Object

The model containing the current column the results should sort on. This directive will watch that column for changes, and request new results from the paginated API, on change

[sortDirection]Object

The model containing the current direction of the current sort column. This directive will watch for changes, and request new results from the paginated API, on change

[errorMessage]String

An error message that should be displayed if a call to the request fails
